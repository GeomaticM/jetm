JETM Springframework integration

Due to the nature of Spring managed applications JETM performance monitoring can easily be added without touching application source code at all. This is done using Spring AOP. With it's help it is possible to monitor execution times purely by configuration.

Therefore JETM comes with a simple and easy to use Springframework integration. The following page lists prerequisites and constraints and refers to the detailed integration documentation.

Prerequisites and constraints

Springframework 1.x or 2.x?

Even though JETM supports both Springframework 1.x and 2.x the integration itself slightly differs. This is because of the newly introduced configuration namespace support in Springframework 2.x. If you are free to choose your Springframework version we recommend using Springframework 2.x, otherwhise you need use the old 1.x style JETM configuration.

Featurewhise there is no difference between Springframework 2.x and 1.x, however in 2.x the configuration is greatly simplified.

Classpath Setup

JETM depends on Springframework classes and it's dependencies. It is necessary to add the following jars to the application classpath.

Dependency JAR File(s)
Spring Framework spring.jar (Spring 1.x, 2.x) or
spring-aop.jar, spring-beans.jar, spring-context.jar, spring-core.jar (Spring 1.x)
AOP-Alliance API aopalliance.jar
CGLIB cglib-nodep-x.jar
Commons-Logging commons-logging.jar
Using ApplicationContext

Be aware that your spring configuration must be running within an ApplicationContext. Normally this is the case if you create the Spring runtime using a Web ContextLoaderListener or Web ContextLoaderServlet or any ApplicationContext subclass. If you just use a BeanFactory the following examples will not work (Hint: Register BeanPostProcessors - e.g. BeanNameAutoProxyCreator - manually).
