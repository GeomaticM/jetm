Java™ Execution Time Measurement Library
Runtime performance monitoring made easy

RRD4j offline integration

Step I: Prerequisites

In order to use JETM RRD4j support you need to add the RRD4j libs to your classpath. You can download the current RRD4j distribution here. Since RRD4j is a Java™ 5.0 port of JRobin it depends on Java™ 5.0 as runtime. JETM 1.2.0 was tested with RRD4j 2.0.x.

Step II: Enable performance result logging

Write raw data to logfile for further processing

Step III: Import raw data to RRD DB

Use command line to create db and import data java -cp $CLASSPATH etm.contrib.rrd.rrd4j.Rrd4jMain create-db -t ${TEMPLATE} -d ${DB_FILE_NAME} java -cp $CLASSPATH $OPTIONS etm.contrib.rrd.rrd4j.Rrd4jMain import -s "${SOURCE_FILE}" -d "${IMPORT_DEFINITION}"

Step IV: Create image

Use command line to create db and import data $JAVA -cp $CLASSPATH ${OPTIONS} etm.contrib.rrd.rrd4j.Rrd4jMain create-graph -t max-average -s ${RRD_FILE} \ -d ${IMAGE_PATH}/performance-year.png -b 2007.01.01 -p imagetitle="xxx"
